## 一、用户原始想法（原汁记录）

**核心洞察**：各业务模块可以有差异化结构，即时零售不必照搬通用模板。

### 竞对动态 → 分两层
**第一层 · O2O 竞对**：2 个固定卡片位
- 美团买药、淘宝闪购买药
- 每个卡片重点关注其各种核心动态

**第二层 · 线下药店竞对**：3 个头部连锁药店卡片
- 老百姓、益丰、大参林
- 每个卡片更新各自动态：财报、高管变动、各类核心情报
- **每个连锁药店卡片上方加一个"财报小卡片"**（常驻、季度更新）：
  - 25年：营收、净利润、利润率、门店数（直营/加盟分别多少）
  - 26年Q1：对应数据
  - 这个数据常驻展示，等后续季度披露了再替换

### 政策动态 → 纳入外卖大盘
- 即时零售与外卖强相关（外卖是大盘），外卖监管/新规/建议也放进来，因为影响大盘

### 信源（用户初步想到）
- 36氪、晚点 LatePost、创业邦

## 二、Claude 评估与补充

### 整体评价
这个"卡片化/实体化"思路**比通用条目列表更适合即时零售**——因为这个赛道的竞对是**固定的几个玩家**（不像新品那样源源不断），用"固定卡片追踪每个对手"比"每天抓零散新闻"更稳定、更有沉淀感。**赞成做。**

### 结构建议（细化用户想法）
```
即时零售
├ 竞对动态
│  ├ 第一层 O2O（2张固定卡）
│  │   ├ 美团买药：核心动态(订单量/扩品类/即时零售战略/补贴)
│  │   └ 淘宝闪购买药：核心动态(阿里整合/闪购医药/饿了么协同)
│  └ 第二层 线下连锁（3张固定卡，每张含"财报小卡"+动态流）
│      ├ 老百姓：[财报小卡] + 动态(高管/并购/门店扩张/O2O布局)
│      ├ 益丰：    [财报小卡] + 动态
│      └ 大参林：  [财报小卡] + 动态
├ 政策动态（医药O2O监管 + 外卖大盘监管）
├ 新技术·新模式（无人化/智慧药房/前置仓/即时配送）
└ 行业·资本（O2O赛道融资/大盘数据/新玩家）
```

### 补充建议（用户没提但值得加的）
1. **O2O 竞对可考虑补第3张**：京东自己的"京东买药/小时达"做对照基准？——或单独不放（因为是自家）。**待定，问BP**。
2. **连锁药店其实有第4家常被提**：一心堂（云南为主）。用户选了老百姓/益丰/大参林（确实是A股三大龙头），**建议保留3家聚焦，一心堂作动态里提及即可**。
3. **财报小卡的字段**建议标准化：`营收 / 净利润 / 净利率 / 门店总数(直营X+加盟Y) / 同比增速`，每家同口径才好横向对比。再加一行"数据截止季度"避免看的人误读时效。
4. **外卖大盘**：除了监管政策，外卖三巨头(美团/饿了么/京东外卖)的即时零售战略动作也值得进"行业·资本"或"新模式"——因为医药即时零售是搭在外卖履约网络上的。
5. **差异化呈现**：这个部门若卡片常驻，前端要改 data.json 结构（加卡片/财报小卡字段）和 index.html 渲染。**属于 P1 范畴，和"滚动情报池"一起评估**。

## 三、推广价值：各部门差异化结构

用户这个想法点出一个更大的方向——**不是所有部门都该用同一个模板**。可借鉴：
- **即时零售**：固定玩家 → 卡片化追踪（本设计）
- **医药/营养/器械/消费器械**：新品源源不断 → 维持条目流 + 新品方法论(SEARCH_GUIDE)
- **消费医疗**：可能也适合"机构卡片"(爱康/美年/新氧)+ 项目流的混合

这个"按业务特性定制模块结构"的思路，建议作为整体优化的一条原则。

## 四、实现记录（2026-06-24）

**已落地内容：**
1. **前端 `index.html`**：`renderBrief` 增加 `b.layout==="instant"` 分支；新增 `renderLayers()`（O2O层2卡 + 连锁层3卡）和 `renderFin()`（财报小卡多期横向对比）。其余 5 部门走原条目流逻辑，**完全向后兼容**（已验证 nutri/consumer 仍为 item 列表、0 卡片）。配套 CSS：`.ccard / .fin / .layer-lbl / .cc-feed` 等。
2. **数据 `data.json`**：instant 改为新结构——O2O 2卡（美团买药/淘宝闪购买药，不放京东）+ 连锁 3卡（老百姓 603883.SH / 益丰 603939.SH / 大参林 603233.SH，各带 2025年报+2026Q1 财报小卡）。财报数字与动态均由子 agent 实抓核实，带真实 url；推算项（益丰直营/加盟拆分）已标注。
3. **`validate.py`**：新增 `check_layers()` 校验卡片结构（type/title/cards、财报 periods.label、卡内动态流字段与日期倒序）；instant 竞对走此分支，其余部门不变。自检全过。
4. **`AGENTS.md`**：数据契约段落新增「即时零售（instant）专属结构」权威说明（含老百姓代码 603883 的纠错提醒）。

**踩到的真实问题（已修正）：**
- 子 agent 诚实纠错：老百姓代码应为 **603883.SH**（002424 是贵州百灵）。
- validate.py 抓出 2 张卡的动态流日期非倒序 → 已用脚本重排。

**决策（用户拍板）：** O2O 不放京东自家（情报台定位对外、自家数据内部系统更准）；本轮范围 = 前端结构先行，数据用现有+实抓填充。
