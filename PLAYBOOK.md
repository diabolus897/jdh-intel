# 抓取剧本（PLAYBOOK.md）

> 每日情报抓取的**可复用执行剧本**。固化"怎么派子agent、每个agent带什么prompt、抓完怎么自检"，避免每轮重新设计。
> 配套：`AGENTS.md`(部门口径) + `SOURCES.md`(信源地图) + `validate.py`(自检脚本)。
> 最后更新：2026-06-24

---

## 一、整体流程（SOP）

```
1. 读 AGENTS.md + PROJECT_STATUS.md + SOURCES.md（带上下文）
2. 决定本轮范围：单部门重抓 / 全量6部门刷新
3. 按「子agent切分策略」并行派抓取agent，每个带全套铁律prompt
4. 汇总 → 跨源去重 → 按时效铁律剔除超期 → 按维度归类
5. hi/mid 重点条目逐条 fetch 原文核确切日期，消除"待核"
6. 用 Python 写入 data.json（覆盖对应部门），不手改JSON避免格式错
7. python3 validate.py 自检，必须全过
8. 本地 preview 渲染确认（可选但推荐）
9. 更新 PROJECT_STATUS.md 第5节状态表 + SOURCES.md 实测标记
10. git commit + push → Netlify 自动部署（约1分钟）
11. 出自检报告：各维度条数、空维度原因、剔除/待核项、失败信源
```

---

## 二、子 agent 切分策略

**经验最优切分**（consumer 那轮验证，6并行≈5分钟，互不阻塞）：

| 切法 | 适用 |
|---|---|
| **按品类深抓 ×N** | 该部门有几个 GMV 大品类，每个 1 个 agent 深抓全维度 |
| **跨品类竞对 ×1** | 平台级竞对动作（大促战报/类目策略），跨所有品类一个 agent |
| **跨品类政策 ×1** | NMPA/医保/市监 监管动态，跨所有品类一个 agent |
| **补充品类 ×1** | 料稀薄的小品类合并到 1 个 agent，抓到几条算几条 |

**火力分配原则**：料密集的维度（竞对/政策）多分 agent；料稀薄的品类（如康复理疗）合并少分。不在低产维度浪费算力。

## 三、子 agent 的标准 prompt 模板

> 派每个抓取子 agent 时，用下面这段打底，再替换【】里的部门/品类/信源。

```
你是京东健康【部门名】部门的竞争情报分析师。今天是【YYYY-MM-DD】。
请联网抓取【品类/维度】的真实、近期、权威情报。

## 铁律（必须严格遵守）
1. 只报本次真实抓取到、有可点开 url 的内容；绝不编造事件/数字/公司名/产品名/url。
   url 必须是抓取到的真实原始链接。
2. 时效分级：竞对/财报/重大BD收购/政策 放宽近30天（约【30天前日期】之后）；
   新品/普通行业动态 守近7天（约【7天前日期】之后）。超期旧闻直接丢弃，不要保留。
3. 日期：抓到精确日期填 "YYYY-MM-DD"，抓不到填 "待核" 不猜；重点条目再 fetch 原文确认。
4. 严格去重：同一公司同一件事只留信息最全的一条。
5. 新品维度只收"具体新品/新型号/新成分/新获批"；趋势/盘点/榜单/展会预告/选购指南一律不收。
6. 反软文：丢弃"十大/测评/排行/推荐"类营销内容。

## 抓取范围
【品牌方清单 / 新品 / 竞对平台 / 政策面，参考 SOURCES.md 对应品类】

## 工具经验（来自 SOURCES.md）
- web_search 把"2026年X月"判成未来而拒答时，去年份改"近期/最新/618"重试。
- web_search 只返回泛化话术时，改用 WebFetch 抓 html.duckduckgo.com/html/?q= 检索页。
- 品牌官网新闻页/Bing 常是 JS 空壳，换垂直媒体或官方公告。

## 输出格式
按维度返回真实条目，每条含：维度、title、rel(hi/mid/lo)、summary(≤60字)、
source、date、url(真实)、takeaway(对JDH采销/商分的一句具体动作提示)。
没抓到合格内容的维度明说"无合格内容"。报告实际抓了哪些源、哪些失败。不编造凑数。
```

---

## 四、写入 data.json 的注意事项

- **用 Python 脚本写，不手改 JSON**：避免逗号/引号/超长 summary 等低级错误。
- summary **必须 ≤60 字**（validate.py 会卡），写之前自己数一下。
- 维度顺序：前5部门「竞对/政策/工业/新品」；**instant 是「竞对/政策/新技术·新模式/行业·资本」**。
- 维度内 items 按 date 从新到旧；无料维度 `items: []`。
- musts 跨维度挑最重要 3 条（优先 hi）。
- 改完务必 `python3 validate.py` 全过再提交。
